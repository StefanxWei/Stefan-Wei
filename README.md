## Synap

I'm currently building **Synap**, a text-processing tool for people who communicate frequently, work across languages, or want to express themselves more clearly.

Today, translating or rewriting a message usually means copying it into a translator or LLM, writing a prompt, waiting for the result, and pasting it back. Synap reduces that process to a small floating bubble.

It can:

* capture, process, and replace text in one flow
* translate or rewrite text without interrupting the current workflow
* adapt to a selected tone, language, and text context
* work across different apps and text fields
* support different LLM providers through BYOK

The first prototype is already working. It includes a structured internal architecture, a minimal interface, local API key handling, and measures designed to reduce workflow interruption and protect user privacy.

I built it using **VMware, Visual Studio, and Codex**.

## Why I Built It

I can usually understand English, but expressing exactly what I mean is harder.

I wanted a tool that could improve or translate text directly inside a conversation, without constantly switching between apps. I couldn't find one that felt simple, direct, and broadly compatible, so I started building Synap.

Early testers found the translation flow fast and convenient. Their main feedback was that the first-use experience needs clearer guidance, which is now one of my next priorities.
